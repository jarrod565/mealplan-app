## Problem

When Aldi can't fulfill an item — a refund notification arrives while the user is in the pickup lane — the user currently has to manually look up that item and add it to a separate Meijer list by hand. This happens in the parking lot, on a phone, under time pressure, immediately before walking into Meijer. It is the most friction-heavy moment in the entire weekly grocery workflow.

The Meijer list also starts incomplete — it only grows as Aldi failures come in, so there's no single ready-to-use list when the user walks into the store.

---

## Outcome

The Meijer list is complete and ready before the user leaves home. When Aldi refunds an item in the pickup lane, one tap moves it to the Meijer list instantly. The user walks into Meijer with a live, accurate list already in hand.

Once implemented:

- The Meijer list is generated as part of CB_14 and is fully populated before the shopping trip begins
- A dedicated "Aldi Pickup" mode on the Meijer list tab makes it fast and one-handed to mark items as unavailable
- Marking an item unavailable in Aldi Pickup mode instantly moves it to the top of the Meijer list
- The Meijer list updates in real time — no refresh required
- Both users on the subscription see the same live list simultaneously
- The Meijer list is usable as a standard in-store checklist once at Meijer

---

## Users

**Primary users:**
- Household member sitting in the Aldi pickup lane marking refunded items
- Household member walking through Meijer checking items off

---

## System Context

The Meijer Live List is the in-trip layer on top of the Meijer list generated by CB_14. It does not generate the list — it makes the existing list reactive to real-world events during the shopping trip.

**Flow:**
```
Weekly lists generated (CB_14) — Meijer list already populated
       ↓
User arrives at Aldi for pickup
       ↓
User opens Dinder → Meijer tab → taps "Aldi Pickup Mode"
       ↓
Aldi Pickup Mode displays the Aldi list items only
       ↓
User receives refund notification from Aldi app
       ↓
User taps the refunded item in Dinder → "Mark as unavailable at Aldi"
       ↓
Item moves instantly to top of Meijer list under "From Aldi" section
       ↓
User taps "Done with Aldi" → Aldi Pickup Mode exits
       ↓
User walks into Meijer — Meijer list is complete and up to date
       ↓
User checks off items as they shop
```

---

## Responsibilities

**This capability is responsible for:**
- Providing an "Aldi Pickup Mode" on the Meijer list tab
- Displaying Aldi list items in Aldi Pickup Mode for easy unavailability marking
- Moving marked-unavailable items to the top of the Meijer list instantly
- Real-time sync of the Meijer list across both subscription users via Supabase Realtime
- Distinguishing "From Aldi" items from planned Meijer items on the list
- Standard in-store checklist behavior (check off, persist check state)

**This capability is NOT responsible for:**
- Generating the Meijer list (CB_14)
- Communicating with the Aldi app or Aldi's servers — unavailability is marked manually by the user
- Pushing the Meijer list to any external app
- Handling Sam's Club fallback (out of scope for v1)

---

## Functional Rules

### Aldi Pickup Mode

- Aldi Pickup Mode is activated by a button on the Meijer tab: "Aldi Pickup Mode"
- In Aldi Pickup Mode, the view shows only the current Aldi list items — not the Meijer list
- Each Aldi item has a single tap action: "Unavailable" — large tap target, usable one-handed
- Tapping "Unavailable" on an item immediately moves it to the Meijer list under a "From Aldi" section at the top
- The item is removed from the Aldi list display in Pickup Mode simultaneously
- Aldi Pickup Mode is exited by tapping "Done with Aldi" or navigating away
- Aldi Pickup Mode can be re-entered at any time during the trip if more refunds come in

### Meijer List During Pickup Mode

- The Meijer list continues to update in real time as items are marked unavailable at Aldi
- The "From Aldi" section always appears at the top of the Meijer list, above planned Meijer items
- Items in the "From Aldi" section are visually distinguished (e.g. a small Aldi indicator badge)
- Both users on the subscription see "From Aldi" items appear in real time via Supabase Realtime

### In-Store Checklist

- Once at Meijer, the user checks off items as they shop — same behavior as the existing Shopping List (CB_07)
- Check state persists across app opens
- "From Aldi" items and planned Meijer items are checked off the same way — no behavioral difference
- When all Meijer items are checked, the user is prompted: "All done at Meijer! Mark as complete?"

### Real-Time Sync

- Supabase Realtime subscription on the `weekly_lists` row for the Meijer store
- Both users see item additions (from Aldi fallout) and check-offs in real time
- No manual refresh required at any point during the trip

---

## Edge Cases

- **User marks an item unavailable then finds it at Aldi** — an "Undo" option is available for 10 seconds after marking unavailable. After that, the item must be manually moved back (tap and hold → "Return to Aldi list"). This is an edge case and not a primary flow.
- **Refund notification arrives after Aldi Pickup Mode is exited** — user can re-enter Aldi Pickup Mode at any time. There is no time limit.
- **Both users marking items simultaneously** — Supabase Realtime handles concurrent updates. Last write wins on individual item state. No conflict UI in v1.
- **No Aldi items were refunded** — Aldi Pickup Mode is used and exited with no items marked. The Meijer list is unchanged from what was generated in CB_14. This is the ideal outcome.
- **User skips Aldi Pickup Mode entirely** — they can still check items off the Meijer list normally. Aldi Pickup Mode is an enhancement, not a required step.

---

## Not In Scope (v1)

- Automatic detection of Aldi refund notifications (requires Aldi API access — not available)
- Sam's Club fallback handling from Aldi or Meijer unavailability
- Sharing the Meijer list with someone not on the subscription
- Navigation / aisle routing within Meijer

## AI Implementation Context

### Architecture

- This capability adds no new Supabase tables — it operates on the `weekly_lists` row for `store: 'meijer'` created by CB_14
- Aldi fallout items are added to the Meijer list's `items` JSONB array with `source: 'aldi_fallout'`
- Supabase Realtime watches the `weekly_lists` row for the current subscription and week
- The "From Aldi" section is rendered by filtering `items` where `source === 'aldi_fallout'`
- Aldi Pickup Mode is a client-side UI state — not persisted to Supabase
- The undo window (10 seconds) is a client-side timer — no server involvement

### Key Implementation Rules

- Realtime subscription should be established when the Meijer tab is first opened, not on app load
- The "Unavailable" tap action in Aldi Pickup Mode must feel instantaneous — optimistic UI update first, Supabase write second
- Aldi Pickup Mode should be designed for one-handed use on a phone in a car — large tap targets, minimal cognitive load, high contrast
- The "From Aldi" badge should use the existing badge component already in the project
- Migration number: none — uses CB_14's tables
